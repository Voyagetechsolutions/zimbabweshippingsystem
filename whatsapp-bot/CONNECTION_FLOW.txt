┌──────────────────────────────────────────────────────────────────────┐
│  📊 DATA FLOW                                                        │
└──────────────────────────────────────────────────────────────────────┘

  When Customer Sends Message:
  
  1. WhatsApp → Bot Server
     ├─ Message text
     ├─ Sender phone number
     └─ Timestamp
  
  2. Bot Server → Database
     ├─ Load user session
     ├─ Get bot settings
     └─ Get bot messages
  
  3. Bot Server (Processing)
     ├─ Identify current step
     ├─ Validate input
     ├─ Calculate pricing (if needed)
     └─ Generate response
  
  4. Bot Server → Database
     ├─ Update user session
     ├─ Save booking data
     └─ Log interaction
  
  5. Bot Server → WhatsApp
     └─ Send response message
  
  6. WhatsApp → Customer
     └─ Display message

┌──────────────────────────────────────────────────────────────────────┐
│  💾 SESSION PERSISTENCE                                              │
└──────────────────────────────────────────────────────────────────────┘

  How Session Data is Saved:
  
  Server Filesystem:
  /opt/zimship-bot/
  ├── whatsapp-session/
  │   ├── creds.json          ← Authentication credentials
  │   ├── app-state-sync-*.json  ← WhatsApp state
  │   └── session-*.json      ← Session data
  ├── qr-code.png             ← QR code image (temporary)
  └── src/                    ← Bot code
  
  Database (Supabase):
  ├── user_sessions           ← Customer conversation state
  ├── bookings                ← Completed bookings
  ├── bot_settings            ← Pricing configuration
  └── bot_messages            ← Bot response templates

  Think of it like this:
  
  1. You rent a computer in the cloud (DigitalOcean)
  2. You install your bot on that computer
  3. You connect your customer's WhatsApp to the bot (QR scan)
  4. The bot runs 24/7 on that computer
  5. When customers message, the bot responds automatically
  6. All data is saved to your database
  
  That's it! Simple, right? 🎉
